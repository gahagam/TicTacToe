# Крестики-нолики (Tic-Tac-Toe)

Мобильное приложение для Android, реализующее классическую игру «Крестики-нолики» на поле 3×3.  
Проект выполнен с акцентом на аналитический подход: формализацию требований, моделирование состояний и документирование логики.

## О проекте

- **Платформа:** Android
- **Язык разработки:** Kotlin
- **Архитектура:** MVC с разделением логики и представления
- **Управление сборкой:** Gradle (Kotlin DSL)

## Аналитическая составляющая

В рамках проекта выполнены следующие аналитические задачи:

### 1. Формализация требований
- Определены функциональные требования:
  - Игровое поле 3×3 с возможностью размещения крестика или нолика
  - Поочередные ходы двух игроков
  - Автоматическая проверка победы после каждого хода
  - Определение ничьей при заполнении всех ячеек
  - Возможность начала новой игры без перезапуска приложения
- Выделены нефункциональные требования:
  - Интуитивно понятный интерфейс
  - Мгновенная реакция на действия пользователя

### 2. Моделирование состояний
Разработана и реализована модель конечного автомата (Finite State Machine) для управления игровым процессом:

| Состояние | Описание | Доступные переходы |
|-----------|----------|-------------------|
| `GAME_SETUP` | Инициализация игрового поля | → `PLAYER_TURN` |
| `PLAYER_TURN` | Ожидание хода игрока | → `CHECK_WIN` |
| `CHECK_WIN` | Проверка победы или ничьей | → `GAME_OVER_WIN` / `GAME_OVER_DRAW` / `PLAYER_TURN` (смена игрока) |
| `GAME_OVER_WIN` | Победа одного из игроков | → `GAME_SETUP` (новая игра) |
| `GAME_OVER_DRAW` | Ничья | → `GAME_SETUP` (новая игра) |

### 3. Описание вариантов использования 

#### Основной сценарий (счастливый путь)
1. Игрок запускает приложение → поле пустое, ход за «X»
2. Игрок нажимает на свободную ячейку → в ячейке отображается «X»
3. Система проверяет победу → победитель не обнаружен
4. Ход переходит к игроку «O»
5. Шаги 2–4 повторяются до достижения победы или ничьей
6. При достижении конечного состояния отображается результат и кнопка «Новая игра»

#### Альтернативные сценарии
- **Попытка хода в занятую ячейку:** система игнорирует действие, ход не меняется
- **Наступление победы:** система блокирует дальнейшие ходы
- **Наступление ничьей:** система объявляет ничью, поле блокируется
